# Program Blueprint Schema

Use this schema when Program Pal designs a training program before creating the final workbook.

## Required Sections

### Client Snapshot

Briefly identify the client context that matters for programming:

- Age or training context when relevant
- Training background
- Primary constraints
- Key preferences
- Important safety flags

### Primary Goal

State the main goal the program is built around.

### Secondary Goals

List supporting goals that should influence exercise selection, session goals, or progression.

### Program Duration

State the program length and explain why it fits the SMART goal timeline or trainer request.

### Session Length

State whether sessions are 30 or 60 minutes.

If session length is unknown, ask before finalizing the blueprint.

### Difficulty Rating

State the target difficulty rating on a 1-10 scale, with 10 being the most challenging. Explain how that rating affects exercise selection, volume, intensity, rest, and progression.

If difficulty rating is unknown, ask before finalizing the blueprint.

### Training Frequency

State the number of training sessions per week.

### Chosen Split

Name the split and explain why it best supports the client goals, schedule, and assessment findings.

### Phase Structure

Describe the phase structure.

Use periodized phases when useful. Use a simpler repeated weekly structure when maintenance, confidence, general fitness, or consistency is the main goal.

### Weekly Structure

List the weekly sessions in order.

Each session must include:

- Session name
- Overarching session goal
- Short rationale
- Warm Up focus, including 1-2 movements that specifically prepare the main workout
- Workout focus
- Cooldown focus

### Session Architecture

Every session must follow:

1. Warm Up
2. Workout
3. Cooldown

The contents inside each section should be determined by the assessment, not by a fixed template.

Warm-ups should include dynamic stretching or light mobility plus 1-2 specific prep movements tied to the session goal, first major workout movement, or client assessment findings.

### Exercise Selection Logic

Explain how exercises are chosen.

Include:

- How exercises support the client goals
- How preferences and dislikes are handled
- How pain, medical constraints, or movement limitations affect choices
- Why any corrective or medical circumstance exercises are placed where they are

### Progression Plan

Explain how the program progresses.

Progression may involve load, reps, sets, range of motion, control, skill quality, conditioning duration, conditioning intensity, or reduced assistance.

### Conditioning Plan

Describe whether conditioning is included, where it appears, and why.

Conditioning can appear early when it is the highest-priority challenge and heavy lifting is not the main session demand.

### Mobility And Flexibility Plan

Describe how mobility and flexibility goals are addressed.

Non-weighted or very light mobility and flexibility movements should be placed primarily in the warm-up or cooldown. Warm-ups and cooldowns should include dynamic stretching.

### Corrective And Posture Strategy

Describe how posture deviations, imbalances, or corrective goals are addressed.

Weighted corrective or posture-focused movements belong in the workout.

### Safety Constraints

List safety constraints and how the program respects them.

### Success Metrics

Define what will show progress.

Examples:

- Better FMS score
- Improved control
- Increased range of motion
- More reps at a target quality
- Higher load at the same technique quality
- Improved conditioning tolerance
- Reduced symptoms
- Better symmetry

### Ready For Workbook

State one of:

- `Yes`: Enough information exists to create the workbook.
- `No`: Missing information must be answered first.

If `No`, list the missing questions.
